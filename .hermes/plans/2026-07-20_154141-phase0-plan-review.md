Blocking issues found and resolved:

1. Test path originally targeted root-level `tests/test_action_latent_statistics.py`. The repository pytest config primarily discovers package tests under `packages/**`, and explicit root tests can interact badly with root coverage settings. Corrected the plan to use `acs-jepa-cli/tests/test_action_latent_statistics.py` and explicit pytest commands for that path.
2. `schema-vs-argument variance decomposition` was originally phrased as optional where the Phase 0 spec requires it. Corrected the plan to require the metric or an explicit `not_available_reason` if a future implementation cannot compute it.
3. Same-schema nearest-wrong margins needed to include the true/reference action latent to nearest wrong same-schema candidate per transition, not only generic candidate-to-candidate nearest-neighbor distances. Corrected the plan and implementation scope to include reference-action margins.
4. Cross-checkpoint comparison across baseline, inverse-dynamics, and RNN-action-encoder checkpoints is required by Phase 0 when artifacts are available. The plan now explicitly treats this first slice as the reusable diagnostic and defers cross-checkpoint reporting until local checkpoints/data exist.
